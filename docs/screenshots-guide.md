# スクリーンショット撮影ガイド

Play Store のスクリーンショット要件:
- **形式**: PNG または JPG
- **比率**: 16:9 縦 (例 1080×1920) または 16:9 横
- **枚数**: 最低 2 枚、最大 8 枚(**4-8 枚が推奨**)
- **サイズ**: 320px〜3840px

## 📷 おすすめ 6 枚

Pixel 7 / Galaxy / iPhone のいずれかの実機で撮影。Expo Go の URL バーや時刻表示は問題ありません。

### 1. セッション一覧画面 (ホーム)
**ストーリー**: 「複数の分析セッションを一括管理」

**操作**:
1. ホーム画面を開く
2. **2-3 件のセッションをあらかじめ作成しておく**(ライン1朝・ライン1夜・新人A、など名前を変えて)
3. スクリーンショット撮影

**ポイント**: カードの並びと FAB が見えること。

---

### 2. 新規セッション作成画面
**ストーリー**: 「ステップ式の簡単セットアップ」

**操作**:
1. + 新規ボタンをタップ
2. 「① 何を分析しますか?」のモードカードが見える画面で撮影
3. **モード「人＋機械」を選択した状態**にすると見栄えが良い

**ポイント**: ① ② ③… のステップ番号と、モードカードが視覚的に分かること。

---

### 3. 作業記録画面 (動画 + 編集パネル)
**ストーリー**: 「動画を見ながらワンタップで記録」

**操作**:
1. 既存セッションを開く → 記録タブ
2. 動画を中央あたりまでシークしてポーズ
3. 「ここから」「ここまで」を実際にセットして、作業名を「組付」など入れた状態で撮影

**ポイント**: 動画 + 入力 UI + 過去の記録リストが**ぜんぶ写る**ことが理想。
画面が縦に長い場合は、入力 UI が中心にくるようにスクロール位置を調整。

---

### 4. ガントチャート画面
**ストーリー**: 「複数リソースを並べた時系列分析」

**操作**:
1. 分析タブ → ガントチャート
2. **複数のリソースに作業を記録した状態**で表示
3. ピンチアウトで適切なズーム

**ポイント**: タクト線(赤破線)と複数行のバーが見えること。

---

### 5. 改善ポイント画面 (シミュレーション含む)
**ストーリー**: 「AI が改善案を自動提案 + 年間効果を試算」

**操作**:
1. 改善タブを開く
2. **設定タブで時給と1日サイクル数を入れておく**(例: 2500/h, 800 台/日)
3. 改善ポイントを 2-3 件タップして選択した状態
4. 画面下に「年間効果見込み ¥XXX万円/年」の緑カードが表示された状態で撮影

**ポイント**: これが最も**売れる**画面。緑カードの金額が大きく見えるようにスクロール位置を調整。

---

### 6. PDF レポート (任意 / 出力サンプル)
**ストーリー**: 「経営層に提出できる本格レポート」

**操作 (オプションA - スマホ画面)**:
1. PDF 出力後にプレビューが開く画面
2. もしくは PDF を別アプリで開いた状態

**操作 (オプションB - PCで撮影)**:
1. 実際に出力された PDF を PC で開く
2. Adobe Acrobat や Chrome の PDF ビューアでサマリページを表示
3. PCのスクショを撮影 → スマホ画面用にリサイズ(または PDF プレビュー画面の合成)

**ポイント**: 円グラフ + KPI + 改善効果カードが見える 1 ページ目を見せる。

---

## 撮影時のチェックリスト

- [ ] ステータスバー(時刻・電池・電波)が**フル表示**になっているか
- [ ] **暗いコンテンツ**(ヘッダー)と**明るいコンテンツ**(ボディ)のコントラストが出ているか
- [ ] サンプルのセッション名・作業名は**それっぽい現場の名前**になっているか
  - ✓「ライン1 朝礼後計測」「組付」「品質チェック」
  - ✗「テスト」「aaa」「あいうえお」
- [ ] 改善ポイントが**実際に検出された状態**で記録しているか(空っぽだと意味不明)
- [ ] 時給設定があるか(改善画面で効果見込みカードが出ている状態か)

## 撮影後の処理

### スマホ実機の場合
- Pixel/Galaxy であれば撮ったスクショ画像をそのままアップロード OK
- ファイルサイズが大きすぎたら(8MB 超) Squoosh.app などで圧縮

### Android Studio エミュレータの場合
- AVD で起動 → サイドメニューのカメラアイコン
- 1080×1920 などの解像度で出力

### スクリーンショット枠の合成 (任意)
**より売れる**スクショにしたい場合は MockUPhone (https://mockuphone.com) などで実機モックアップに埋め込むと一段プロっぽくなります。

## 推奨フローまとめ

```
1. ライン1朝計測 セッションを作る (5分)
   ・モード「人＋機械」
   ・作業者2人、機械1台
   ・タクトタイム 60秒
   ・動画読み込み

2. 5-6 件の要素作業を記録 (10分)
   ・正味・付帯・ムダをバランスよく
   ・タクト超過するくらいの長さに

3. 設定 → 改善効果試算
   ・時給 2500
   ・1日 800 台
   ・年間 250 日

4. 順番にスクリーンショットを撮る (10分)
   1) ホーム → ⓘで メニュー閉じておく
   2) 新規セッション(モード選択画面)
   3) 記録画面(動画 + 入力)
   4) 分析 → ガントチャート
   5) 改善 → 緑カード表示
   6) PDF 出力プレビュー
```

合計 30 分で全アセットがそろいます。
